What You Need to Build a Website Using Only Your Phone

Building a website on your phone is 100% possible! You don’t need a laptop or expensive tools. Here’s everything you need:

  • 1. A Code Editor App: You need an app to write and save code.
    • **Acode (Android):** Powerful and easy to use.
    • **Spck Editor (Android):** Great for GitHub users.
    • **Kodex (iOS):: If you use an iPhone.
  • 2. A Browser: Use your phone’s browser (like Chrome or Safari) to preview your website.
  • 3. File Manager (Optional): To organize your files (HTML, images, CSS, etc.).
  • 4. GitHub App (Optional): For publishing your website online.
    • Create an account on GitHub.
    • Upload your index.html file to a new repository.
    • Use GitHub Pages to make your site live for free.
  • 5. Internet Connection: Needed only when uploading your site or installing apps. You can write code offline.

🚀 Quick Guide: How to Make a New Repository and Go Live

  1. Install a code editor like **Spck Editor** from the Play Store and open it.
  2. Create a **GitHub** account (if you don't have one) and then create a new repository.
  3. In your code editor (e.g., Spck Editor), start a new project and create an `index.html` file.
  4. Link your project with your GitHub repository through the Git settings in your code editor.
  5. **Commit** your changes (save them to your Git history) and then **Push** them to GitHub.
  6. Go to your GitHub repository on the web, navigate to **Settings** > **Pages**, and **Enable GitHub Pages** for your `main` branch.
  7. Your site will be live at: `https://yourusername.github.io` (replace `yourusername` with your actual GitHub username).

This guide is meant to show that anyone can create a modern website using only a smartphone. Just follow the steps above and explore — it’s easier than you think!

Step-by-Step: Create a GitHub Repository & Publish Your Website

GitHub repositories store a variety of projects. In this guide, you'll learn how to create a repository, add your website files, and then publish your site using GitHub Pages.

1. Create a New Repository

Visit: https://github.com/new

In the upper-right corner of any GitHub page, select the '+' icon (or your profile dropdown), then click **New repository**.

2. Name Your Repository and Add Details

  • For **Repository name**, enter: `yourusername.github.io` (e.g., `kuuriyawep.github.io`). This specific naming convention is crucial for GitHub Pages to automatically serve your website.
  • For **Description**, optionally add: "My personal website built with HTML"
  • Under **Public / Private**, select: **Public** (required for GitHub Pages)
  • Under **Initialize**, check: `[✔] Add a README file` (optional, but good practice for project description)

3. Create Repository

Click the **"Create repository"** button.

Congratulations! You've successfully created your first repository.

---

How to Add `index.html` Code Inside the New Repository

1. Go to your New Repository

Navigate to your newly created repository. For example, if you named it `kuuriyawep.github.io`, go to: https://github.com/kuuriyawep/kuuriyawep.github.io

2. Add a New File (`index.html`)

  1. Click: **Add file**
  2. Select: **Create new file**
  3. Type the filename as: `index.html` (this is the default file GitHub Pages looks for to display your website).

3. Write Your HTML Code

In the text editor, paste or type your HTML code. This `index.html` file will be the main page of your website.



  
    My Awesome Mobile Website
  
  
    
Hello, World! This is my first website built on a phone!

    
I can add paragraphs, bold text, and links.

    
A Subheading

    
Learning to code on a phone is really exciting.

4. Save the File (Commit Changes)

  1. Scroll down to the "Commit changes" section.
  2. In the **Commit message** field, write a short, meaningful message like: `Add index.html` or `Initial website setup`.
  3. Click the **"Commit new file"** button.

This saves your `index.html` file to your repository's `main` branch.

How to Publish Your Website Using GitHub Pages

1. Open Repository Settings for Pages

Go to your repository's settings for GitHub Pages. You can find this by navigating to your repository (e.g., `https://github.com/kuuriyawep/kuuriyawep.github.io`), then clicking on **Settings** in the top navigation bar, and then **Pages** from the left sidebar. Alternatively, use this direct link: https://github.com/kuuriyawep/kuuriyawep.github.io/settings/pages

2. Set up GitHub Pages Source

  1. Under **"Build and deployment"**, find the **"Source"** section.
  2. From the dropdown menu, select: `main branch`
  3. Then choose: `/ (root)`
  4. Click: **Save**

3. Get Your Website Link

After a few seconds, GitHub Pages will deploy your website. You'll see a message confirming that "Your site is live at...". Your site will typically be live at:

https://kuuriyawep.github.io (replace `kuuriyawep` with your GitHub username)

It might take a few minutes for the site to become fully accessible after deployment. If you don't see it immediately, give it a few minutes and refresh the page.

Frequently Asked Questions (FAQ)

Q: Do I need to pay anything to build a website using this guide?

A: No, absolutely not! All the tools and methods described in this guide, including hosting on GitHub Pages, are completely free to use. This entire website was built without any cost.

Q: How long does it take to build my first website?

A: The basic steps to get your first HTML page live can be completed in just a few hours. Learning more advanced concepts like CSS and JavaScript will take more time, but the initial setup is very quick.

Q: Can I add more advanced features like CSS or JavaScript?

A: Yes! Once you understand the basics of HTML and how to publish, you can easily add CSS files to style your website and JavaScript files to add interactive features. Just upload them to your GitHub repository alongside your `index.html`.

Q: What if I don't have an Android phone?

A: No problem! The guide includes `Kodex` for iOS users. While the specific steps for code editors might differ slightly, the core principles of writing HTML and using GitHub remain the same across platforms.

What's Next? Continue Your Web Development Journey!

Congratulations on building your first website with just your phone! This is a huge accomplishment. But the journey doesn't stop here. Here are some next steps to deepen your web development skills:

  • Learn CSS for Styling: Your website needs to look good! CSS (Cascading Style Sheets) is used to control the colors, fonts, layout, and overall appearance of your web pages.
  • Master JavaScript for Interactivity: Bring your website to life with JavaScript. This language allows you to add dynamic features, animations, form validations, and much more.
  • Build Multi-Page Websites: Instead of just one `index.html`, learn how to link multiple HTML pages together to create a full website with different sections (e.g., About, Services, Contact).
  • Explore Responsive Design: Make sure your website looks great on all devices, from small phones to large desktop screens. This is crucial for a modern web presence.
  • Consider a Custom Domain: While `yourusername.github.io` is free, you can eventually connect a custom domain name (like `yourwebsite.com`) to your GitHub Pages site for a more professional look (this usually involves a small cost for the domain name itself).

Keep experimenting, keep learning, and don't be afraid to break things and fix them. That's how you truly learn!
